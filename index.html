BIOS 學習筆記

  • Power ON：電腦通電後，CPU 從 BIOS 開始執行。
  • POST：執行加電自我測試，檢查記憶體、鍵盤、顯示卡等硬體。
  • 啟動順序：根據設定尋找開機裝置（如 SSD、USB）。
  • 載入 bootloader：如 GRUB、Windows Boot Manager。
  • 交棒給作業系統：開始進入 OS 運作。
  • 早期 BIOS 儲存在 ROM：穩定但無法更新，需更換整顆晶片。
  • 現代 BIOS 使用 Flash Memory：可重寫的 EEPROM，如 SPI Flash、NOR Flash。
  • 更新方式：
    • 廠商軟體（如 ASUS EZ Flash、MSI M-Flash）
    • BIOS 內建工具更新
    • USB 開機更新（需 FAT32 格式）
  • 更新風險：斷電或版本錯誤會導致主機板無法啟動。
  • 建議更新時機：新增 CPU 相容性、修正安全性漏洞、有明確異常時。
練習題：為何現代 BIOS 要使用 Flash 儲存？
答：因為 Flash 可重寫、容量大，更適合現代系統的更新需求。

📌 Day 3 將介紹 BIOS 與 CMOS 的關係
  • BIOS：是一段儲存在主機板上的韌體，用來在電腦開機時初始化硬體與啟動作業系統。
  • CMOS：是一塊使用電池供電的記憶體，用來保存 BIOS 設定值（如開機順序、時間、電壓等）。
  • 兩者關係：
    • BIOS 是程式、CMOS 是儲存設定的地方。
    • 你在 BIOS 畫面設定的內容（如時間、開機順序），實際儲存在 CMOS。
  • CMOS 電池沒電會怎樣？
    • 時間會歸零
    • BIOS 設定會被重設為預設值
  • 啟動流程中誰先執行？ → CPU 先執行 BIOS → BIOS 從 CMOS 讀取設定 → 執行 POST → 找開機裝置
練習題：BIOS 和 CMOS 有什麼不同？哪個會因電池沒電失效？
答：BIOS 是執行程式，不會因沒電消失；CMOS 是設定資料，會因電池沒電而失效。

📌 持續更新中：Day 4 將介紹 POST 詳細流程與錯誤訊號（Beep code）
  • POST： Power-On Self-Test 是 BIOS 加電後進行的自我檢查流程。
  • 檢查項目： CPU、記憶體、顯示卡、鍵盤、滑鼠、CMOS 設定與開機裝置。
  • Beep Code（嗶嗶聲）： 若畫面無輸出，BIOS 會用聲音提示錯誤。
    • 1 短嗶：正常開機
    • 2 短嗶：記憶體問題
    • 3 長嗶：鍵盤未接
    • 5 短嗶：主機板錯誤
    • 無聲：CPU 或主機板可能損壞
  • 提醒： 各主機板 Beep Code 不同，請查詢官方手冊。
練習題：POST 與 BIOS beep code 有什麼關係？
答：POST 是 BIOS 開機流程的一部分，若檢查出錯，BIOS 會透過 beep code 傳達錯誤類型。

📌 Day 5 預告：進入 BIOS 畫面後可以設定哪些參數？（如開機順序、XMP、AHCI）

📊 Beep Code 對照表

嗶聲次數 / 範例 常見意義（AMI BIOS） 處理建議
1 短嗶 正常開機 無需處理
2 短嗶 記憶體錯誤 重新插拔記憶體 / 更換記憶體
3 長嗶 鍵盤未接 檢查鍵盤連接線
5 短嗶 主機板錯誤 檢查主機板狀況或送修
無聲 CPU 或主機板損壞 檢查 CPU 安裝與主機板通電狀況
  • Boot Priority：決定從哪個裝置開機（SSD / USB / 光碟機）。
  • SATA Mode（AHCI）：用來設定硬碟通訊方式，建議使用 AHCI 模式。
  • XMP 記憶體設定：套用記憶體最佳化頻率（如 DDR4-3200）。
  • Secure Boot：啟動系統安全驗證，避免惡意軟體進入開機流程。
練習題：Secure Boot 有什麼作用？關掉會怎樣？
答：Secure Boot 能保護系統避免執行非授權開機檔案，但也可能導致無法安裝自訂系統（如 Linux）。

📌 Day 6 將介紹 BIOS 實際操作畫面模擬與 POST 路徑。
  • Power ON：接通電源後，CPU 從 BIOS 開始執行。
  • POST：執行加電自我測試，檢查 CPU、記憶體、顯示卡等基本硬體。
  • 硬體初始化：偵測鍵盤、滑鼠、硬碟等，設定環境。
  • 讀取 CMOS：讀取使用者設定，例如開機順序、時間、Secure Boot 狀態。
  • 載入 bootloader：找出開機裝置，將控制權交給 bootloader。
  • 交棒給 OS：載入作業系統核心，完成 BIOS 任務。
練習題：BIOS 的任務到哪裡結束？什麼時候 CMOS 設定會生效？
答：BIOS 任務在交棒給 bootloader 後結束，CMOS 設定在初始化階段會被讀取並應用。

🗓️ BIOS 學習筆記：第 7 天(2025/6/14)

📌 主題：BIOS 的基本作業與工作流程

BIOS（Basic Input/Output System） 是嵌入在主機板上的韌體，是所有電腦一開機最先被執行的程式。

🔸 BIOS 的五大核心作業：

  • 1️⃣ 上電自檢（POST）： 開機瞬間執行的檢查程序，確認 CPU、RAM、顯示卡等基本硬體正常。
  • 2️⃣ 載入 CMOS 設定： 讀取 RTC 中的設定，例如開機順序與系統時間。
  • 3️⃣ 初始化硬體： 啟動各項裝置如 USB、SATA、PCI，準備系統環境。
  • 4️⃣ 搜尋開機裝置： 根據設定順序讀取裝置的 bootloader。
  • 5️⃣ 交棒作業系統： 將控制權交給 bootloader，啟動作業系統核心。

📌 主題：進階 BIOS 設定與風險

  • Fast Boot：略過部份 POST，加速開機，可能錯過 BIOS 熱鍵。
  • VT-x / AMD-V：開啟虛擬化支援，必要於 VirtualBox / WSL2。
  • CSM：相容傳統 BIOS，若系統為 UEFI + GPT 建議關閉。
  • TPM：信賴平台模組，用於 Windows 11 憑證與 BitLocker。

⚠️ 常見風險對照：

  • 改錯開機模式：可能導致 OS 無法啟動
  • 關閉 TPM：BitLocker 資料可能被鎖住
  • 開啟 Fast Boot：難以進入 BIOS 設定

🧪 練習題：

Q：你想安裝模擬器 / Docker，該開什麼？

A：請開啟 BIOS 的 VT-x 或 AMD-V。


📌 預告：Day 9 將介紹 BIOS 密碼設定與清除方式

📌 主題：BIOS 密碼保護與清除方式

  • Supervisor Password： 管理者密碼，限制進入 BIOS 設定畫面。
  • User Password： 開機密碼，防止未授權者啟動作業系統。

🔓 清除 BIOS 密碼的方法：

  • 🧩 拔除 CMOS 電池等待放電
  • 🧩 使用主機板跳線（JCMOS）清除
  • 🧩 使用廠商提供的密碼恢復工具

🧪 練習題：

請問 Supervisor 與 User 密碼差在哪？怎樣強制清除？

答：Supervisor 控 BIOS 設定權限，User 控開機；可透過拔電池 / 跳線清除。


📌 預告：Day 10 將介紹 UEFI 與 GPT / MBR 開機架構的差異

📌 主題：UEFI 與 BIOS 的差異與開機架構

  • BIOS：傳統韌體架構，支援 MBR 格式磁區，開機程序較慢且功能有限。
  • UEFI：現代主機板使用的新架構，支援 GPT 格式，可啟用 Secure Boot、快速開機等功能。

🔄 開機流程差異：

項目 BIOS + MBR UEFI + GPT
開機磁區 MBR（最多 4 個主分割區） GPT（支援 128 個分割區）
啟動方式 從 Boot Sector 跳轉至 OS 直接執行 EFI 檔案
支援容量 最大 2TB 最大 9.4ZB（實際依硬碟限制）
圖形介面 無（文字為主） 支援滑鼠、圖形 BIOS 設定

🛡️ Secure Boot 是什麼？

Secure Boot 是 UEFI 功能之一，能驗證開機用的 EFI 檔案是否經過數位簽章，避免開機過程被植入惡意軟體。

🧪 練習題：

Q：為什麼新系統建議使用 GPT + UEFI 而非 MBR？

A：GPT 支援更多分割區與大容量硬碟，而 UEFI 開機更快、更安全（支援 Secure Boot），比傳統 BIOS 更適合現代作業系統。


📌 預告：Day 11 將介紹 BIOS 韌體更新與常見風險（Flash BIOS 的原理）

📌 主題：BIOS 韌體更新（Flash BIOS）與常見風險

  • BIOS 韌體：寫在主機板上的非揮發性儲存記憶體（如 EEPROM / Flash ROM），控制開機流程與硬體初始化。
  • 為何需要更新：
    • ➕ 新硬體支援（如新款 CPU）
    • 🔒 安全性修補
    • 🧩 修正開機 bug 或相容性問題
    • ⚙️ 增加功能（如 XMP、TPM 支援）

🔄 BIOS 更新方式

  • BIOS 內建工具：如 ASUS EZ Flash / MSI M-Flash，可直接從 USB 更新
  • Windows 工具：從桌面執行更新程式（風險較高）
  • USB 開機更新：製作 FAT32 格式隨身碟，搭配更新工具啟動

⚠️ 常見風險

  • 🔋 斷電：更新中斷電會導致 BIOS 損毀，主機板無法啟動
  • 🌀 錯誤檔案：若更新錯誤型號，可能導致系統啟動失敗
  • 💾 格式問題：USB 建議使用 FAT32 格式，否則 BIOS 可能無法辨識
  • 🛑 中途關機或重啟：會導致 BIOS 寫入不完整，造成主機板錯誤

✅ 更新前建議

  • 確認主機板型號與 BIOS 檔案完全符合
  • 使用穩定電源，避免過程中有震動或斷電
  • 更新完建議進入 BIOS 載入 Default 設定再儲存退出
  • 優先使用 BIOS 內建工具更新（安全性較高）

🧪 練習題

Q：如果更新 BIOS 時斷電怎麼辦？

A：若主機板有 Dual BIOS 會自動還原，否則可：

  • 試著啟動主機板提供的「BIOS 恢復模式」
  • 拔掉 CMOS 電池等待放電再重設
  • 送修或請專業人員使用燒錄器重新寫入 BIOS

一開始接觸文學的機會是在國三的「音樂課」，對，音樂。 我們必須要做一份報告，有關於音樂劇《悲慘世界》。在找尋資料的過程中我發現他是有原著的，於是， 我去圖書館找來看，厚度可以拿來敲暈別人，但，我讀到淚流滿面。 此時，對於文學的愛意種子發芽。

漸漸地我開始從雨果去搜尋其他作品，當我回過神時我的書櫃已經變成文學的產地。 但我不滿足於此，所有經過翻譯的文字是會變形的，不論信達雅做得多好。 故我將手伸向原文，因為我的英文能力，我可以從更深層的地方感受到這些作者為何使用這些詞語， 為何如此編排情緒與句構，所以也可以說是我對文學的熱愛推進了我學習語言的部分原因。

如同我後來因為一些遊戲接觸到大正文學，日文的最終目標，就是成為那些大師身旁的讀者，跨越時空與背景， 這是讀文學最讓人感動的，如同《蘭亭集序》縮寫，我們，會站在跟那些留下印記的文人身邊，感受與理解他們的思想。

最喜歡的莫過於《基督山恩仇記》，他是個很好閱讀，屬於大仲馬時代的八點檔，但文筆流暢，即使經過翻譯還是能體會他的文學風格與浪漫。

大正文學則是我最崇敬的芥川龍之介老師，他的孤寂與對自身的要求，非常的痛苦細膩。 我曾經在讀他的全集時，他又表明過，他的遺書並不想被公開，但，後代世人還是將其當作他的文學， 我心一橫，將書中那個篇章損毀，並燃燒，這是我對於作者最高敬意的表現，謝謝您，芥川老師。 最喜歡的代表作是短篇：《海市蜃樓》。

P.S. 所以其實應該寫成「西洋經典文學」和「日本大正文學」，更加精準。

西洋，為何是西洋呢，因為也跟我一開始碰到的文學有關係，我開始認識了作者雨果，他所撰寫的《悲慘世界》也是真實巴黎公社的一段。我自然而然開始去參考他的背景，逐漸地發現原來所有的一切，科學文學藝術文化都是建立在「文明」也就是「歷史」之上。

最有趣發生過的是音樂課！老師正在介紹音樂家，突然跳到了神聖羅馬帝國、普魯士等，老師也說他沒研究。我就自告奮勇開始講解羅馬帝國史，一直講到一戰、二戰與東西德。

所以，其實歷史影響著不只是歷史，而是所有的你對一切的認知與生活。後來的歷史課，我的課本上都是額外的補充知識，彷彿在看一場人類如何走到如今的紀錄片，而不是為了考試而讀。

人必須發展出社會三科的穩定才有辦法在生活安穩的基礎上發展文明科學甚至思想！
故沒有文理孰重孰輕，這些基礎才是我們該尊重的，沒有他們也就沒有後來的二三四類未來。
